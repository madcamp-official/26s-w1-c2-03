> 최종 수정: 2026-07-03
> 팀: 2인 (프론트엔드 / 백엔드 분담)

---

## 1. 한 줄 정의

맛집·카페를 직접 방문하고 **음식 사진으로 인증**해 **뱃지를 모으고 랭킹을 정복**하는
포켓몬고식 오프라인 탐험 게임. 사장님에겐 단골을 게임처럼 만들어주는 **B2B 마케팅 도구**.

**서비스명 (확정)**: 맛짱 (Matzzang)

---

## 2. 등장인물과 인센티브

| 대상 | 왜 쓰는가 |
|---|---|
| **유저(손님)** | ① 방문이 게임이 됨(뱃지 수집·랭킹 경쟁) ② 열심히 다니면 진짜 리워드(사장님 제공) ③ "나의 미식 정복 지도" 기록의 즐거움 |
| **사장님** | ① 단골이 자발적으로 자주 오게 만드는 게임화된 마케팅 ② 상위 방문자에게만 리워드를 걸어 충성 고객을 저비용 보상 ③ 신규 손님 유입 |
| **운영(우리)** | 매장 DB 선점 → 유행 → 사장님 유입(B2B 수익화) |

---

## 3. 핵심 루프 (유저 경험)

```
지도에서 근처 맛집 발견 → 방문 → 앱 카메라로 음식 사진 촬영
   → 사장님에게 인증 요청 전송 (대기중)
   → 사장님이 카운터에서 [수락] 탭
   → ① 즉시: 스탬프 +1, 뱃지 진행, 랭킹 반영 ("지금 쌓이는 맛")
   → ② 월말: 방문 랭킹 상위 유저에게 사장님 리워드 자동 지급 ("1등 하면 큰 보상")
   → 리뷰까지 남기면 '리뷰왕' 칭호 추가
```

포켓몬고의 "체육관 점령 → 유지 → 보상"과 같은 구조.

---

## 4. 4개의 핵심 시스템

### (A) 방문 인증 — 사장님 승인 방식
- 매장에서 **앱 내 카메라**로 음식 사진 촬영 (갤러리 업로드 지양)
- 사장님에게 인증 요청 전송 → 사장님 대시보드에 실시간 알림
- **기본 방식: 카운터 즉석 수락** — 손님이 계산 시 화면을 보여주면 사장님이 그 자리에서 탭
- 보조: 한가할 때 밀린 요청을 리스트로 모아 일괄 승인
- 수락 시 방문 확정 → 스탬프/뱃지/랭킹 반영, 거절 시 무효

### (B) 뱃지 시스템 (수집욕 자극)
- **매장 뱃지**: 특정 매장 N회 방문 (예: 단골 뱃지 10회)
- **카테고리 뱃지**: "카페 20곳 정복", "국밥 마스터", "매운맛 헌터"
- **지역 뱃지**: "성수동 정복", "홍대 완주"
- **활동 뱃지**: "리뷰왕", "얼리버드", "탐험가(신규 매장 첫 인증)"

### (C) 랭킹
- **매장별 랭킹**: 매장 방문 횟수 상위 유저 (→ 사장님이 리워드 대상 확인)
- **카테고리/전체 랭킹**: "이 달의 카페 최다 방문자" 등

### (D) 리워드 — 혼합형
- **① 즉시 스탬프**: 수락 시마다 적립, N개 모으면 쿠폰 (즉각적 재미)
- **② 월별 랭킹 리워드**: 매달 1일 지난달 랭킹 확정 → 상위 유저에게 사장님 등록 리워드 자동 지급
- 뱃지별 상위 랭커는 카테고리 맞춤 리워드

## 5. 리스크와 해법 (악마의 변호인)

| 리스크 | 해법 |
|---|---|
| **방문 인증 조작** (사진 도용·GPS 조작) | 사장님 승인으로 사람이 직접 검증 → 조작을 눈으로 걸러냄. 100% 방어보다 '게임/마케팅 힌트'로 포지셔닝 |
| **사장님 승인 피로** (러시 타임) | 카운터 즉석 수락 / 일괄 승인 / (나중)단골 자동승인 |
| **콜드 스타트** (사장님 없으면 리워드 없음) | 초기엔 사장님 없이도 뱃지·정복지도·전체 랭킹 자체가 재미. 리워드는 사장님이 붙는 보너스 |
| **네이버 리뷰 API 부재** | 네이버 대신 자체 리뷰로 '리뷰왕' 칭호 운영 (데이터도 우리 소유) |
| **혼합형 리워드 범위 과다** | 단계 쪼개기: 즉시 스탬프 먼저 완성, 월별 자동지급은 여유 시 |

## 6. 경쟁 서비스 분석

| 서비스 | 하는 일 | 우리와 차이 |
|---|---|---|
| **포스퀘어/스웜** (해외, 원조) | 체크인·뱃지·메이어(매장 1위) | 재미 구조는 검증됨. 단, '체크인만'으론 질림 → 우리는 리워드·정복지도로 보완. **사장님 리워드 자동화는 우리 강점** |
| **다이닝코드** | 맛집 검색·랭킹·리뷰 | 검색·리뷰 중심. 뱃지 게임·사장님 랭킹 리워드 없음 |
| **캐치테이블** | 맛집 예약 | 게임 아님 |
| **도도포인트** | 전화번호 적립 CRM | 지도·뱃지·게임 없음 |

→ **국내엔 "뱃지 수집 게임 + 사장님 랭킹 리워드"를 정면으로 하는 서비스가 없음 = 우리의 빈틈**

## 7. 일주일 MVP 범위

### 완성 순서
1. **1순위 (반드시)**: 사진 인증 → 사장님 즉석 수락 → 즉시 스탬프/뱃지/랭킹. 이 루프 하나가 끝까지 돌면 데모 성립.
2. **2순위 (되면)**: 월별 랭킹 리워드 자동 지급 (배치 로직).

### ✅ 넣기 vs ⏸️ 나중
| ✅ MVP | ⏸️ 나중 |
|---|---|
| 지도에서 매장 보기 (사장님 등록 매장) | 정교한 어뷰징 탐지 |
| 앱 카메라 사진 인증 | 단골 자동승인 |
| 사장님 승인 화면 (핵심!) | 카카오/네이버 로그인(초기엔 간단 식별) |
| 즉시 스탬프/뱃지/랭킹 | 리워드 실물/쿠폰 정산 자동화 |
| "나의 정복 지도" 개인 페이지 | 결제/유료화 |
| 사장님 리워드 등록 화면 | |

## 8. 성공의 척도 (데모)

- 심사위원이 그 자리에서 링크/QR 접속 → 사진 인증 → 사장님 수락 → 뱃지 획득 → 랭킹에 이름 오름을 **직접 체험**
- "내 정복 지도가 채워지는" 시각적 쾌감
- 실시간(사장님 수락 → 손님 화면 즉시 반응) 연출

---

## 9. 확정된 핵심 결정

- **인증 방식**: 사장님 승인제 (카운터 즉석 수락 기본)
- **리워드**: 혼합형 (즉시 스탬프 + 월별 랭킹 리워드)
- **리뷰**: 자체 리뷰 기반 '리뷰왕'
- **매장 등록**: 사장님이 직접 등록(이름·주소·카테고리·키워드). 주소→좌표는 카카오 API 자동 변환. (실출시 시 선등록+claim 하이브리드 검토)
